1. Lead with value

Start with the most useful thing: the answer, insight, recommendation, confirmation, or limitation.

Ask: What does the user need to know first?

2. Make the next step clear

Every response should help the user continue with a path, CTA, review step, follow-up, or support route.

Ask: What should the user do next?

3. Keep the user in control

When AI prepares or applies something, the user should understand what will happen and be able to review before changes are made.

Ask: Does the user need to review or confirm this?

4. Build trust when it matters

Use data, sources, context, previews, or fallback when the response affects decisions, cost, settings, support, or account-specific outcomes.

Ask: What does the user need to trust this?

Common Response Parts

Not every response needs every part. The structure should match the user's intent and risk level.

Main Message
The primary answer, insight, confirmation, or limitation.
Guidance
Path, steps, instructions, or supporting detail.
Supporting source / data
Evidence that helps users trust the response.
Action
CTA, deep link, review step, or next move.
Fallback / Escalation
Safe path when AI cannot complete or resolve the request.
Feedback
User rating or response-quality signal.

Clarification

Narrows broad, ambiguous, or incomplete requests before choosing a response pattern.

Use when

  • The user's request could go in multiple directions
  • Required information is missing
  • The AI needs the user to choose a path before responding
01 Acknowledge intent
02 Ask clarifying question
03 Offer options
Examples

Clarification is needed when the request could lead to multiple paths:

Improve shipping costs
Fix this issue
Create a rule
Help with labels
Optimize my workflow

Capability Boundary

Explains when AI cannot complete a request and provides the safest next step.

Use when

· AI cannot perform the requested action
· AI does not have access to the required data or system
· The request requires human support
· The action is outside the product's current AI capability
· The safest path is to guide, route, or escalate instead
01 Acknowledge intent
02 State the boundary
03 Offer next step

Escalation

Routes the user to Help Center when the issue cannot be safely or confidently resolved.

Use when

· The issue remains unresolved after guidance or retry
· The request involves billing, refunds, compliance, or sensitive changes
· The user is blocked and cannot continue
· The user explicitly asks for support
01 Acknowledge intent
02 Explain why escalation is needed
03 Route to support

Prompt Chips

Suggested follow-up queries that help users continue the conversation.

Example
When to use
  • After clarification responses to offer specific paths
  • After limitation messages to suggest related queries
  • To guide users toward actionable next questions
Guidelines
  • Keep text concise (3-5 words max)
  • Use sentence case
  • Offer 2-4 options maximum
  • Make each option distinct and actionable

Buttons & CTAs

Action buttons that navigate users to specific pages or trigger system actions.

Example
When to use
  • For deep links to specific pages or features
  • For support handoffs and escalations
  • For capability boundary routing
  • When the action takes the user outside the chat
Guidelines
  • Use action verbs (Go to, View, Contact, Get)
  • Keep text clear and destination-focused
  • Use buttons instead of prompt chips for navigation
  • Limit to 1-3 button options per response

Clarification Card

A card component that visually represents a clarification flow — used when AI needs to narrow an ambiguous request before choosing a response pattern.

Example
When to use
  • The user's request could go in multiple directions
  • Required information is missing before AI can respond
  • The AI needs the user to choose a path before proceeding
Guidelines
  • Ask one clarifying question — not multiple at once
  • Offer 2–4 concrete options when possible
  • Acknowledge the user's intent before asking
  • Keep the question short and focused
